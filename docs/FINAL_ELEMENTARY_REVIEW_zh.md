# 本科级最终复核说明

## 复核结论

当前公开仓库已经达到本科级“仿真系统 + 数据链路 + 基础分类验证 + 文档封装”的交付目标。它包含 Geant4 C++ 仿真代码、材料目录、批量配置、W 靶 120 kV 能谱、事件级 CSV 输出、Python 特征工程脚本、训练/测试拆分、验证证据包、图表、论文式正文和组员学习指南。

最重要的复核结论是：项目已经形成闭环，但闭环范围是仿真条件下的粗粒度吸收组分类验证。它不是工业设备验证，也不是所有矿物覆盖证明。

补充复核结论是：材料级分选诊断已经启动，但当前结果未通过验收。旧公开数据上的十材料直接分类主方法 top-1 accuracy 为 `0.464`，top-3 accuracy 为 `0.876`，macro-F1 为 `0.4486`，review rate 为 `0.848`。因此不能把 `0.9960` 二分类结果说成十材料物种识别结果。

## 已完成证据链

```mermaid
flowchart LR
    A["材料目录与配置"] --> B["Geant4 仿真"]
    B --> C["events.csv / hits.csv"]
    C --> D["数据质量检查"]
    D --> E["虚拟样本与特征"]
    E --> F["训练/测试拆分"]
    F --> G["阈值法与 Logistic Regression"]
    G --> H["混淆矩阵、manifest、论文"]
```

`results/undergrad_validation/` 是当前最重要的复核入口。它显示十种材料均有 5000 个仿真事件；每 100 个事件聚合为一个虚拟样本；总计 500 个虚拟样本；训练集 250 个样本，测试集 250 个样本。当前证据包中，阈值法、单特征 Logistic Regression 和三特征 Logistic Regression 的测试 accuracy 分别为 `0.9840`、`0.9920`、`0.9960`。

## 可以成立的表述

可以说：本项目完成了基于 Geant4 的 XRT 矿物分选仿真原型，能够输出事件级数据，并用 Python 将仿真数据转化为可解释特征和本科级分类验证结果。

可以说：在当前十种单一材料、固定几何、仿真数据和低/高吸收组二分类任务中，三特征 Logistic Regression 在 250 个测试虚拟样本上正确 249 个，测试 accuracy 为 `0.9960`。

可以说：当前分类结果来自明确的训练/测试拆分，不是训练集准确率。

可以说：项目当前价值在于把物理仿真、数据生成、特征工程、基础模型、结果证据和文档组织成可学习、可运行、可复查的本科工程研究闭环。

## 不应成立的表述

不应说：项目已经验证真实 XRT 设备效果。

不应说：模型能识别世界上所有矿物。

不应说：当前模型已经能准确识别十种材料。材料级诊断结果已经显示旧数据不足以支持这个结论。

不应说：当前结果可以直接用于工业在线控制。

不应说：`0.9960` 是普适常数。Geant4 仿真具有随机性，重新生成事件 CSV 后结果可能小幅波动。

不应说：当前项目已经完成产品级自动分选。真实产品还需要真实样本、设备标定、混合矿流、实时性、误分成本、执行机构和人工复核流程等证据。

## 后续改进方向

如果要继续提高研究等级，建议优先补充固定随机种子或多随机种子重复实验、不同厚度和粒径配置、混合材料样本、按独立 run 拆分的测试集、交叉验证，以及真实设备数据对照。

如果要探索产品方向，建议先做“物理/化学特征输出 + 矿物字典候选检索 + 人工复核”的 MVP，而不是直接宣称模型可以自动输出分选动作。
