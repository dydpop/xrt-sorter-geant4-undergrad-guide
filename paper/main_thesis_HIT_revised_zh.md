# 基于 Geant4 的 X 射线透射矿物分选仿真系统设计与本科级验证

## 摘要

X 射线透射分选利用不同材料对 X 射线衰减能力的差异，为矿物预选和物料识别提供了一种非接触式检测思路。由于真实 XRT 设备、标准样品和现场运行数据获取成本较高，本项目以本科研究训练和工程实现为目标，构建了一个基于 Geant4 的 XRT 矿物分选仿真原型。系统使用 C++ 实现 X 射线源、矿物样本、探测器、事件记录和命中记录，使用配置文件控制材料、几何和源项参数，并通过 Python 对仿真事件数据进行样本构造、特征提取和基础分类验证。公开复现实验包含 Quartz、Calcite、Orthoclase、Albite、Dolomite、Pyrite、Hematite、Magnetite、Chalcopyrite 和 Galena 十种单一材料，每种材料运行 5000 个仿真事件，每 100 个事件聚合为 1 个虚拟样本，共得到 500 个虚拟样本。按每种材料前 25 个样本训练、后 25 个样本测试的规则，形成 250 个训练样本和 250 个测试样本。结果显示，基于主 gamma 透射率的阈值法在测试集上达到 0.9840 accuracy，单特征 Logistic Regression 达到 0.9920，三特征 Logistic Regression 达到 0.9960。该结果说明当前仿真链路能够生成具有可分性的物理相关特征，并支持低吸收组与高吸收组的本科级粗粒度分类验证。本文结论仅限于当前十材料、固定几何、仿真数据和二分类任务，不代表真实设备性能、复杂矿流表现或对所有矿物的普适分类能力。

**关键词**：Geant4；X 射线透射；矿物分选；仿真；Logistic Regression；本科项目

## 1. 引言

矿物分选的核心问题之一，是在破碎、磨矿或进一步加工之前尽可能识别有价值物料和低价值物料，从而降低后续处理成本并提高资源利用效率。传统分选方法往往依赖密度、磁性、颜色、粒度或人工经验，但复杂矿物体系中，外观相近而内部组成不同的物料并不容易通过表面信息区分。X 射线透射技术关注 X 射线穿过样本后的强度和能量响应变化。当样本材料的有效原子序数、密度和厚度不同，透射信号会发生不同程度的衰减，因此 XRT 被广泛视为传感器分选中的重要方向之一。

对于本科项目而言，直接使用真实 XRT 分选设备存在明显门槛。真实设备通常需要稳定的射线源、探测器、屏蔽结构、样品输送系统和安全规范；真实矿石数据还受到样品来源、粒径分布、含水率、杂质、仪器校准和现场工况的影响。若在项目早期直接追求现场数据和工业级结论，容易出现证据不足、实验不可复现或结论越界的问题。因此，本项目选择先建立仿真系统，通过可控的材料、几何和源项配置来验证 XRT 信号链路，并把结果限定在本科级系统实现和基础分类验证范围内。

Geant4 是一个用于粒子与物质相互作用模拟的工具包，常用于高能物理、医学物理、空间辐射和探测器建模等场景。它适合本项目的原因在于：X 射线本质上是光子，穿过材料时涉及光电效应、康普顿散射、瑞利散射等电磁过程；Geant4 能够以事件为单位追踪粒子过程，并把命中、能量沉积和几何边界信息输出为可分析数据。本项目在 Geant4 示例框架基础上改造出一个 XRT 仿真程序，并围绕本科论文和组内学习需求整理为公开仓库。

本文的研究目标不是证明某个工业设备已经可用，也不是建立覆盖所有矿物的通用识别模型，而是回答三个本科阶段可以严谨回答的问题。第一，能否用 Geant4 搭建一个包含源项、样本、探测器和事件输出的 XRT 仿真系统。第二，仿真输出能否转化为有物理意义的特征，例如透射率、探测器能量沉积和 gamma 命中率。第三，在明确训练/测试拆分的前提下，这些特征能否支持低吸收组和高吸收组的粗粒度分类验证。

## 2. 相关工作与技术基础

Geant4 的经典论文将其描述为一个面向探测器和粒子输运问题的仿真工具包，其设计特点包括几何建模、材料定义、物理过程、事件跟踪、命中记录和用户动作接口。后续发展工作进一步扩展了电磁过程、强子过程、低能模型和应用范围。对本项目来说，Geant4 的价值并不在于自动给出分类模型，而在于提供一个可编程、可配置、可追踪的物理仿真环境，使 X 射线穿过矿物样本后的响应可以被逐事件记录。

XRT 分选和传感器分选文献通常强调两个层面：一是物理信号层面，即不同材料、不同厚度、不同有效原子序数会影响透射响应；二是决策层面，即如何把传感器信号转化为分选判断。工程系统往往还需要考虑输送速度、颗粒姿态、探测器阵列、喷吹执行机构和现场维护。本项目只覆盖其中的前两个简化层面：通过仿真产生透射相关信号，再用基础算法验证这些信号是否具有分类区分度。

机器学习部分采用了阈值法和 Logistic Regression。阈值法具有很强的可解释性，适合验证主透射率是否已经能区分高吸收与低吸收材料。Logistic Regression 是一种经典线性分类方法，常用于二分类任务；在 scikit-learn 中，`StandardScaler` 可以先对特征进行标准化，`LogisticRegression` 再学习线性判别边界。本项目选择这些基础方法，是因为本科阶段更需要清楚解释特征、样本和边界，而不是使用复杂模型掩盖数据链路本身。

## 3. 系统设计

本项目系统由 Geant4 C++ 仿真、配置输入、Python 分析和结果文档四个部分组成。C++ 部分负责生成物理事件，配置文件负责控制材料、源项和几何参数，Python 部分负责把事件数据转化为样本和分类结果，文档部分负责把代码、数据和结论组织成可复查的项目交付。

![系统流程](../figures/elementary_system_flow.png)

程序入口为 `exampleB1.cc`，核心类位于 `include/` 和 `src/`。`DetectorConstruction` 负责世界体、样本和探测器几何；`PrimaryGeneratorAction` 负责按照配置文件生成 gamma 源项；`SteppingAction` 在粒子步进过程中识别进入探测器的 gamma 命中并记录能量、位置、偏转角和 primary 标记；`EventAction` 在每个事件结束时汇总探测器响应；`RunAction` 在 run 开始和结束时管理 CSV 文件、累计量和 metadata 输出。这个结构接近 Geant4 用户应用的常见组织方式，便于从源项、几何、事件、命中、运行统计几个层面逐步定位问题。

源项采用 `source_models/spectra/w_target_120kV_1mmAl.csv` 中的 W 靶 120 kV 能谱。源项配置保留能谱模式、相空间模式和单能模式，但本科公开复现实验使用能谱模式。程序读取能谱文件后构造累计分布，并在每个事件中按权重采样 photon energy。这样做比固定单一能量更接近 XRT 背景下的宽能谱设定，但仍然属于简化仿真，不包含真实设备全部结构。

样本几何采用单一材料 slab，公开复现实验中样本厚度为 10 mm，探测器位于样本后方。十种材料分别为 Quartz、Calcite、Orthoclase、Albite、Dolomite、Pyrite、Hematite、Magnetite、Chalcopyrite 和 Galena。材料分组不是细粒度矿物识别，而是本科阶段的粗粒度吸收组任务：Quartz、Calcite、Orthoclase、Albite 和 Dolomite 归为低吸收组，Pyrite、Hematite、Magnetite、Chalcopyrite 和 Galena 归为高吸收组。

## 4. 数据采集与字段定义

每次 Geant4 run 会生成事件级 CSV、命中级 CSV 和 metadata。事件级 CSV 是本次分类验证的主要输入，其表头由 `src/RunAction.cc` 写出，包含 `event_id`、`detector_edep_keV`、`detector_gamma_entries` 和 `primary_gamma_entries`。其中 `event_id` 表示模拟事件编号；`detector_edep_keV` 表示该事件中探测器沉积能量；`detector_gamma_entries` 表示 gamma 进入探测器的次数；`primary_gamma_entries` 表示 primary gamma 到达探测器的次数。

命中级 CSV 记录更细的探测器命中信息，包括命中位置、photon energy、是否为 primary、偏转角，以及近似直接透射和散射后透射标记。当前直接/散射判据是工程近似：primary gamma 到达探测器时，若相对束流方向的偏转角小于 1 度，则记为 direct primary，否则记为 scattered primary。该判据有助于本科阶段理解信号来源，但不能被解释为严格的完整散射物理分类。

公开验证证据位于 `results/undergrad_validation/`。`event_row_summary.csv` 记录十种材料的事件行数检查；`absorption_group_virtual_samples.csv` 记录由事件聚合得到的 500 个虚拟样本；`train_test_split_samples.csv` 记录每个虚拟样本属于训练集还是测试集；`feature_group_summary.csv` 和 `material_feature_summary.csv` 记录分组与逐材料特征分布；`absorption_group_classification_summary.csv` 和三个混淆矩阵文件记录分类结果；`validation_manifest.json` 记录生成脚本、样本政策、软件版本和结论边界。

表 2 给出了事件字段到样本特征的关系。

| 样本级变量 | 来源 | 计算方式 | 物理含义 |
| --- | --- | --- | --- |
| `sample_id` | `event_id` | `event_id // 100` | 每 100 个事件形成一个虚拟样本 |
| `mean_detector_edep_keV` | `detector_edep_keV` | 100 个事件能量沉积均值 | 平均探测器能量响应 |
| `detector_gamma_rate` | `detector_gamma_entries` | 探测器 gamma 命中总数 / 100 | 探测器命中率 |
| `primary_transmission_rate` | `primary_gamma_entries` | 主 gamma 命中总数 / 100 | 主 gamma 透射率 |
| `group_label` | 材料分组表 | 材料名映射 | 低吸收组或高吸收组 |

## 5. 样本构造、训练/测试拆分与分类方法

公开复现实验对每种材料运行 5000 个事件。Python 脚本设置 `PHOTONS_PER_SAMPLE = 100`，因此每种材料形成 50 个虚拟样本，十种材料合计 500 个虚拟样本。为了避免训练集评估冒充测试集，脚本按材料分别切分样本：每种材料前 25 个虚拟样本作为训练集，后 25 个虚拟样本作为测试集。因此训练集包含 250 个样本，测试集包含 250 个样本，并且低吸收组和高吸收组在训练集、测试集中都各占 125 个样本。

本项目采用三种基础分类方法。阈值法只使用 `primary_transmission_rate`，先在训练集中计算低吸收组和高吸收组的平均透射率，再取两者中点作为阈值。测试样本透射率高于阈值则判为低吸收组，低于阈值则判为高吸收组。单特征 Logistic Regression 仍只使用 `primary_transmission_rate`，但用 `StandardScaler + LogisticRegression` 学习二分类边界。三特征 Logistic Regression 使用 `primary_transmission_rate`、`mean_detector_edep_keV` 和 `detector_gamma_rate`，用于检验多个物理相关特征组合是否能提高测试集表现。

评价指标采用 accuracy 和 confusion matrix。Accuracy 表示测试样本中预测正确的比例；confusion matrix 用于显示低吸收组和高吸收组分别被预测到哪个类别。本文所有 accuracy 均来自测试集，不使用训练集准确率作为结果报告。

## 6. 结果

十材料复跑后，`event_row_summary.csv` 显示每种材料均生成 5000 个事件，`event_id` 范围为 0 到 4999，未出现重复事件编号；每种材料均形成 50 个完整虚拟样本，没有被丢弃的尾部事件。按分组统计，低吸收组和高吸收组各有 250 个虚拟样本，训练集和测试集各有 250 个虚拟样本。

表 3 给出了当前证据包中的测试集分类结果。

| 方法 | 特征数 | 测试样本数 | 正确样本数 | 测试 accuracy |
| --- | ---: | ---: | ---: | ---: |
| 阈值法 | 1 | 250 | 246 | 0.9840 |
| 单特征 Logistic Regression | 1 | 250 | 248 | 0.9920 |
| 三特征 Logistic Regression | 3 | 250 | 249 | 0.9960 |

![基础分类精度](../figures/elementary_absorption_accuracy.png)

三种方法都表现出较高的测试集 accuracy。阈值法达到 0.9840，说明单独使用主 gamma 透射率已经可以较好地区分当前十材料配置中的低吸收组和高吸收组。单特征 Logistic Regression 正确 248 个测试样本，三特征 Logistic Regression 正确 249 个测试样本，说明在当前证据集中加入能量沉积和 gamma 命中率能够提供轻微增益。

混淆矩阵进一步说明错误类型。三特征 Logistic Regression 的测试集混淆矩阵显示，高吸收组 125 个测试样本全部预测正确，低吸收组 125 个测试样本中有 1 个被判为高吸收组。这个结果符合 XRT 任务中的基本直觉：高吸收材料整体透射率较低，低吸收材料整体透射率较高，但低吸收材料内部仍可能因为随机事件波动出现局部较低的透射率，从而产生少量误判。

特征分布也支持上述解释。`feature_group_summary.csv` 显示，在训练集中低吸收组 `primary_transmission_rate` 均值为 0.31656，高吸收组为 0.03248；在测试集中低吸收组均值为 0.31256，高吸收组为 0.03248。`material_feature_summary.csv` 进一步显示 Galena 的主 gamma 透射率在当前配置下为 0，而 Quartz、Albite 等低吸收材料保持较高透射率。换言之，分类器并不是凭借无物理意义的索引或文件名分类，而是利用了由仿真事件形成的透射响应差异。

## 7. 讨论

本项目的主要贡献在于建立了一个可运行、可复查、可讲解的本科级 XRT 仿真项目链路。与只提交静态报告相比，当前仓库包含 C++ 仿真源码、材料和源项配置、Python 分析脚本、验证证据包、图表、复现说明和论文式文本。组员或导师可以沿着配置文件、事件 CSV、虚拟样本、训练/测试拆分和混淆矩阵逐层复查结论来源。

当前结果之所以能达到较高 accuracy，主要原因是任务被限定为粗粒度吸收组二分类，而且材料设置为十种单一材料 slab。对于低吸收硅酸盐、碳酸盐和高吸收铁氧化物、硫化物之间的区分，透射率本身就具有较强解释力。因此，较高 accuracy 是合理的，但不能被扩大解释为复杂矿流中的细粒度矿物识别能力。

需要特别说明的是，最初六材料已经足够支撑本科级验证，因为本科主线要证明的是 Geant4 仿真、CSV 输出、特征构造、训练/测试拆分和结果证据包这条链路是否成立，而不是证明覆盖所有矿物。当前十材料证据包进一步纳入 Albite、Dolomite、Chalcopyrite 和 Galena，使材料库展示更充分，也说明系统可以通过材料目录、配置文件和重新生成证据包进行扩展。但这仍然不是产品级覆盖：新增材料能否稳定分类，必须以新增后的证据包为准。

本项目存在几个明确限制。第一，数据来自 Geant4 仿真，不是真实 XRT 设备采集数据。第二，训练集和测试集来自同一类仿真配置，只能说明同分布仿真切分下的表现，不能说明跨设备、跨厚度、跨粒径、跨矿区或跨随机种子的泛化能力。第三，样本为单一材料 slab，没有覆盖真实矿石中常见的混合、包裹、裂隙、孔隙和颗粒姿态变化。第四，当前模型只做低吸收组/高吸收组二分类，没有做矿物种类级分类。第五，Geant4 仿真具有随机性，如果不固定随机种子，重新生成事件 CSV 时 accuracy 可能出现小幅波动。

这些限制并不否定项目价值，反而界定了项目的真实完成度。对本科项目来说，合理目标是完成系统搭建、数据链路、特征解释和基础验证，而不是宣称已经达到工程部署条件。后续若要提高研究等级，应优先增加独立重复仿真、不同厚度和粒径配置、混合材料样本、固定随机种子策略、交叉验证或按 run 拆分的独立测试，并最终与真实设备数据进行对照。另一个值得保留的后续方向，是把模型从“直接输出材料标签或动作”转向“输出可解释的物理/化学特征，再结合矿物字典做候选检索和人工复核辅助”；该方向目前只是研究设想，不能写成本科阶段已完成能力。

## 8. 复现与数据可用性

公开仓库提供复现实验所需的代码、配置和结果证据。具备 Geant4、CMake、C++17 编译器、Python、pandas 和 scikit-learn 环境后，可按照 `docs/RUN_LOCALLY_zh.md` 和 `paper/reproducibility.md` 中的命令重新运行十材料配置。运行后，`analysis/classify_absorption_groups.py` 会读取 `build/` 下的十个 `*_events.csv` 文件，并在 `results/undergrad_validation/` 下生成验证证据包。

当前提交的证据包包括事件行数摘要、虚拟样本表、训练/测试拆分表、特征统计、分类汇总、混淆矩阵、测试集预测表和 manifest。为了避免提交本机绝对路径和大型临时文件，`build/` 不作为 Git 跟踪内容；公开仓库跟踪的是紧凑的验证证据和可复现脚本。

## 10. 结论

本文完成了一个基于 Geant4 的 XRT 矿物分选仿真系统设计与本科级验证。系统能够通过材料目录和配置文件运行十种材料的 X 射线透射仿真，输出事件级 CSV，并由 Python 构造虚拟样本、提取物理相关特征、完成明确训练/测试拆分下的粗粒度吸收组分类。当前证据包显示，三特征 Logistic Regression 在 250 个测试虚拟样本上正确 249 个，测试 accuracy 为 0.9960。该结果支持“当前仿真设置下低吸收组与高吸收组具有可区分 XRT 特征”的结论，但不支持真实设备部署、复杂矿流泛化或所有矿物普适分类等更强说法。项目的实际价值在于把仿真、数据、特征、模型、结果和文档组织成了一个可学习、可运行、可复查并可继续扩展材料库的本科工程研究闭环。
