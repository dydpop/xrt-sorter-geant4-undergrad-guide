# English Abstract

This undergraduate project designs and implements a Geant4-based prototype for X-ray transmission (XRT) mineral-sorting simulation. The system uses C++ to model the X-ray source, mineral sample, detector, event records, and detector-hit records. Configuration files define the materials, geometry, and source settings, while a Python workflow converts simulated event data into virtual samples, interpretable features, and baseline classification evidence. The public reproduction package covers six single-material configurations: Quartz, Orthoclase, Calcite, Pyrite, Hematite, and Magnetite. Each material is simulated with 5000 events, and every 100 events are aggregated into one virtual sample, yielding 300 virtual samples in total. For each material, the first 25 samples are used for training and the remaining 25 samples are used for testing, resulting in 150 training samples and 150 test samples. In the current evidence package, a transmission-threshold classifier reaches 0.9800 test accuracy, one-feature Logistic Regression reaches 0.9867, and three-feature Logistic Regression reaches 0.9933. These results show that, within the present six-material simulated setup and coarse absorption-group task, XRT event data can provide discriminative physics-related features. The findings do not represent physical equipment performance, complex ore-stream behavior, or universal classification capability across all minerals.
