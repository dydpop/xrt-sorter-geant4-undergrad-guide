# English Abstract

This undergraduate project designs and implements a Geant4-based prototype for X-ray transmission (XRT) mineral-sorting simulation. The system uses C++ to model the X-ray source, mineral sample, detector, event records, and detector-hit records. Configuration files define the materials, geometry, and source settings, while a Python workflow converts simulated event data into virtual samples, interpretable features, and baseline classification evidence. The public reproduction package covers ten single-material configurations: Quartz, Calcite, Orthoclase, Albite, Dolomite, Pyrite, Hematite, Magnetite, Chalcopyrite, and Galena. Each material is simulated with 5000 events, and every 100 events are aggregated into one virtual sample, yielding 500 virtual samples in total. For each material, the first 25 samples are used for training and the remaining 25 samples are used for testing, resulting in 250 training samples and 250 test samples. In the current evidence package, a transmission-threshold classifier reaches 0.9840 test accuracy, one-feature Logistic Regression reaches 0.9920, and three-feature Logistic Regression reaches 0.9960. These results show that, within the present ten-material simulated setup and coarse absorption-group task, XRT event data can provide discriminative physics-related features. The findings do not represent physical equipment performance, complex ore-stream behavior, or universal classification capability across all minerals.
